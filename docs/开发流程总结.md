> 项目日期：2026-03-09

## 一、项目概述

### 1.1 项目背景
构建户用储能设备 BMS（电池管理系统）的云端监控前端应用，支持设备绑定、实时监控、告警管理和历史数据趋势分析。

### 1.2 核心功能
- 用户认证（登录/注册）
- 设备绑定与管理
- 实时数据监控（电压、温度、SOC 等）
- 告警中心
- 历史数据趋势图

## 四、开发流程

### 4.1 初始化步骤

#### 1. 环境准备
```bash
# 克隆项目
git clone <repo-url> bms_monitor
cd bms_monitor

# 安装依赖
npm install

# 复制环境变量

```

#### 2. 配置 Supabase
1. 创建 Supabase 项目
2. 获取项目 URL 和 Anon Key
3. 编辑 `.env` 文件：
```env
VITE_SUPABASE_URL=https://xxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJxxx
```

#### 3. 初始化数据库
在 Supabase SQL Editor 中运行 `supabase-schema.sql`：
```sql
-- 创建核心表
CREATE TABLE devices (...);
CREATE TABLE user_devices (...);
CREATE TABLE telemetry (...);
CREATE TABLE alerts (...);

-- 启用 RLS 行级安全
ALTER TABLE devices ENABLE ROW LEVEL SECURITY;

-- 创建访问策略
CREATE POLICY "用户可访问绑定设备" ON user_devices ...;
```

## 五、核心模块实现

### 5.1 认证模块（Auth）

**文件**: `src/stores/authStore.ts`, `src/hooks/useAuth.ts`

**实现要点**:
- 使用 Zustand 管理认证状态
- Supabase Auth 处理登录/注册
- `onAuthStateChange` 监听会话变化
- `initializeAuth` 初始化时恢复会话

**关键代码**:
```typescript
export const useAuthStore = create<AuthState>((set, get) => ({
  initializeAuth: async () => {
    const { data: { session } } = await supabase.auth.getSession()
    set({ user: session?.user ?? null, session: session ?? null })

    supabase.auth.onAuthStateChange((_event, session) => {
      set({ user: session?.user ?? null, session: session ?? null })
    })
  }
}))
```

### 5.2 设备管理模块（Device）

**文件**: `src/stores/deviceStore.ts`, `src/hooks/useDevices.ts`

**实现要点**:
- 通过 `user_devices` 关联表查询用户绑定的设备
- 支持设备绑定（bind）和解绑（unbind）
- RLS 策略确保用户只能访问自己的设备

**关键代码**:
```typescript
fetchDevices: async () => {
  const { data, error } = await supabase
    .from('user_devices')
    .select(`
      device_id, role,
      devices:device_id (device_id, manufacturer, status, ...)
    `)
    .order('created_at', { ascending: false })

  const devicesList = (data || [])
    .filter(item => item.devices)
    .map(item => item.devices)

  set({ devices: devicesList })
}
```

### 5.3 告警模块（Alert）

**文件**: `src/stores/alertStore.ts`

**实现要点**:
- 同样通过 `user_devices` 关联查询
- 区分活跃告警（`end_time === null`）和历史告警
- 支持按设备 ID 过滤

### 5.4 布局与路由

**文件**: `src/App.tsx`, `src/components/layout/MainLayout.tsx`

**实现要点**:
- `ProtectedRoute` 组件保护认证路由
- `MainLayout` 提供侧边栏导航和顶部栏
- 响应式侧边栏（可折叠）

## 六、遇到的问题与解决方案

### 问题 1：RLS 策略导致查询返回空数据

**现象**: 登录后 Dashboard 不显示设备数据

**原因**: 直接在 `devices` 表上查询，但 RLS 策略限制了访问

**解决**:
- 通过 `user_devices` 关联表查询
- 使用嵌套查询提取设备详情
```typescript
// 错误方式 - 被 RLS 拦截
await supabase.from('devices').select('*')

// 正确方式 - 通过 user_devices 关联
await supabase
  .from('user_devices')
  .select(`device_id, devices:device_id (...)`)
  .eq('user_id', user.id)
```

### 问题 2：Tailwind CSS v4 配置

**现象**: `grid grid-cols-4` 等 Tailwind 类不生效

**原因**: Tailwind CSS v4 需要显式导入

**解决**: 在 `index.css` 顶部添加：
```css
@import "tailwindcss";
```

### 问题 3：侧边栏文字对比度

**现象**: 深色背景 (#1e3a5f) 下文字看不清

**解决**:
- 标题区域添加渐变背景 `bg-gradient-to-r from-blue-900 to-blue-800`
- 图标和文字统一使用 `text-white`
- 菜单项通过 CSS 覆盖颜色：
```css
.menu-dark-text .ant-menu-item {
  color: rgba(255, 255, 255, 0.85) !important;
}
```

### 问题 4：Ant Design Menu 组件 Props 警告

**现象**: React 警告 `itemClassName` 和 `selectedClassName` 不是有效 props

**解决**: 移除这些 props，改用 CSS 类覆盖样式

## 八、最佳实践

### 8.1 TypeScript 类型安全
- 所有数据库表在 `types/database.ts` 定义类型
- Store、Props、State 全部使用 TypeScript 接口
- API 响应使用统一类型 `ApiResponse<T>`

### 8.2 状态管理
- 使用 Zustand 创建轻量级 Store
- 每个模块独立 Store（auth、device、alert）
- 通过 Custom Hook 封装 Store 逻辑

### 8.3 错误处理
```typescript
try {
  const { data, error } = await supabase.from(...).select(...)
  if (error) throw error
  // 处理数据
} catch (err) {
  set({ error: err instanceof Error ? err.message : '操作失败' })
}
```

### 8.4 响应式设计
- 侧边栏断点响应式折叠
- 卡片网格使用 `grid-cols-4` 响应式布局
- 表格支持移动端滚动

## 十、经验教训

### 10.1 数据库设计
1. **用户 - 设备关联表很重要**: 使用 `user_devices` 表支持多用户共享设备
2. **RLS 策略要仔细测试**: 确保策略不阻止合法查询
3. **外键关联查询**: 使用 Supabase 的嵌套查询语法简化代码

### 10.2 前端开发
1. **Tailwind CSS v4 需要显式导入**: 不要忘记 `@import "tailwindcss"`
2. **Ant Design 组件 Props**: 注意区分组件支持的 props，避免 DOM 警告
3. **深色背景文字对比度**: 至少使用 `text-gray-100` 或 `text-white`

### 10.3 项目管理
1. **环境变量模板**: 使用 `.env.example` 记录必要配置项
2. **数据库 Schema 版本化**: `supabase-schema.sql` 提交到 Git
3. **类型定义集中管理**: `types/database.ts` 作为单一数据源
